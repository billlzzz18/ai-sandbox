## Brief overview
  ชุดกฎสำหรับการพัฒนาโปรเจ็ค AI Agent Framework โดยเน้นการใช้ภาษาไทยในการสื่อสารและการทดสอบอย่างครอบคลุม
## Communication style
  - ตอบเป็นภาษาไทยเสมอเมื่อทำได้ โดยเฉพาะเมื่ออธิบายผลการทำงานหรือสรุปข้อมูล
  - ให้คำอธิบายที่กระชับแต่ครบถ้วน ไม่ verbose เกินไป
  - ใช้ emoji เพื่อช่วยในการอ่าน เช่น ✅ สำหรับความสำเร็จ ❌ สำหรับข้อผิดพลาด
## Development workflow
  - ตรวจสอบความถูกต้องของไฟล์ทั้งหมดก่อน commit โดยใช้ validate.py
  - ทดสอบระบบอย่างครอบคลุมก่อนปล่อยฟีเจอร์ใหม่
  - ใช้ version control อย่างเหมาะสมเมื่อมีการเปลี่ยนแปลงสำคัญ
## Coding best practices
  - ปฏิบัติตาม schema ที่กำหนดไว้สำหรับทุกไฟล์ config
  - เขียนโค้ดให้สามารถนำกลับมาใช้ใหม่ได้ (reusable)
  - จัดระเบียบโฟลเดอร์และไฟล์ให้เป็นระบบ
## Project context
  - โปรเจ็คนี้เป็น AI Agent Framework ที่รองรับหลายภาษาและเครื่องมือ
  - เน้นการทำงานร่วมกับ Gemini CLI และ Q4 framework
  - มีระบบทดสอบที่ครอบคลุมทั้ง unit, integration และ e2e
## Other guidelines
  - อัปเดต version ใน package.json เมื่อมีการเปลี่ยนแปลงสำคัญ
  - ตรวจสอบ import paths ให้ถูกต้องก่อน commit
  - ใช้ markdown สำหรับเอกสารประกอบอย่างสม่ำเสมอ
